# Development Module Descriptions (Updated for Current Version)

The following module descriptions align with the current SENTINEL interface and should replace outdated descriptions that were based on earlier builds.

## Figure 12. User Login Module
The User Login Module is the secure access point of the platform. It authenticates users and routes them to role-based workspaces, ensuring that each account can only access authorized operational functions.

## Figure 13. Superadmin Dashboard Module
The Superadmin Dashboard Module provides command-level visibility over personnel, assets, incidents, and ongoing operations. It consolidates high-priority system status and serves as the control point for global operational decisions.

## Figure 14. Administrator Dashboard Module
The Administrator Dashboard Module supports daily operations management, including coordination of personnel and resource workflows. It is designed for execution and monitoring of operational tasks under administrative scope.

## Figure 15. Supervisor Dashboard Module
The Supervisor Dashboard Module presents field-focused oversight tools for deployment and mission coordination. It enables supervisors to track assigned operations and manage frontline execution status.

## Figure 16. Guard Dashboard Module
The Guard Dashboard Module is the field interface for duty personnel. It emphasizes assignment awareness, rapid status actions, and emergency-ready controls for real-time operational response.

## Figure 17. Approvals Module
The Approvals Module manages account and operational requests that require authorized review. It centralizes decision checkpoints so role-gated actions are approved, deferred, or rejected with accountability.

## Figure 18. Scheduling Module
The Scheduling Module supports shift planning, duty distribution, and workforce assignment control. It helps maintain staffing continuity and aligns personnel schedules with operational requirements.

## Figure 19. Calendar Module
The Calendar Module visualizes schedule and assignment timelines in a date-structured layout. It improves coordination by presenting duty events, planning windows, and scheduling context in one view.

## Figure 20. Operations Map Module
The Operations Map Module provides geospatial awareness of active and planned field operations. It is used to monitor deployment context, location-based activity, and map-centric operational tracking.

## Figure 21. Management Module
The Management Module is the primary workspace for managing user and organizational records. It supports administrative maintenance of account-level data and operational roster information.

## Figure 22. MDR Import Module
The MDR Import Module handles structured ingestion and validation of MDR workbook data. It converts external records into platform-ready entries while preserving import control and review checkpoints.

## Figure 23. Missions Module
The Missions Module organizes mission planning and mission-level execution tracking. It enables supervisors to monitor mission progress and maintain operational continuity across assignments.

## Figure 24. Trips Module
The Trips Module manages trip lifecycle records related to operational movement and deployment support. It captures trip context required for field coordination and post-operation review.

## Figure 25. Inbox Module
The Inbox Module centralizes user-facing operational messages and pending updates. It supports timely awareness of notifications, workflow prompts, and system-generated communication.

## Figure 26. Support Module
The Support Module is the assistance channel for operational and technical concerns. It allows users, especially field personnel, to submit and monitor support requests within the platform.

## Figure 27. Feedback Submission Module
The Feedback Submission Module allows users to submit structured feedback on system usability and operational behavior. It supports continuous improvement through role-based user input.

## Figure 28. Feedback Dashboard Module
The Feedback Dashboard Module aggregates submitted feedback for review and prioritization by leadership roles. It supports quality monitoring and evidence-based refinement decisions.

## Figure 29. Firearms Module
The Firearms Module maintains firearm inventory records and status visibility. It supports accountability and operational readiness checks for controlled equipment.

## Figure 30. Firearm Allocation Module
The Firearm Allocation Module manages issuance and assignment of firearms to authorized personnel. It ensures traceability and controlled allocation across duty workflows.

## Figure 31. Firearm Permits Module
The Firearm Permits Module tracks permit-related records and compliance status. It strengthens regulatory alignment by surfacing permit validity in operational workflows.

## Figure 32. Armored Cars Module
The Armored Cars Module manages armored vehicle profiles and deployment-related information. It supports transport readiness and visibility of mission-critical vehicle resources.

## Figure 33. Maintenance Module
The Maintenance Module records and monitors equipment and vehicle maintenance activities. It supports preventive upkeep and helps reduce operational downtime.

## Figure 34. Analytics Module
The Analytics Module presents performance indicators and resource-utilization summaries for command analysis. It enables data-informed decisions through consolidated operational metrics.

## Figure 35. Audit Module
The Audit Module provides traceability of actions, events, and decision-sensitive operations. It strengthens governance by supporting review, anomaly inspection, and accountability checks.

## Figure 36. Profile Module
The Profile Module manages user account identity details and personal settings relevant to platform use. It allows controlled updates to account-level information.

## Figure 37. Settings Module
The Settings Module contains configurable system and user preferences that affect workflow behavior and interface experience. It supports controlled personalization and environment-level adjustments.
