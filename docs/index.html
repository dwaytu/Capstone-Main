Core Capabilities

Everything Security Operations Needs

SENTINEL consolidates every operational domain — from guard scheduling and firearm inventory to AI-driven absence prediction — into a unified command interface.

🔐

Authentication & RBAC

JWT-based auth with email verification, password recovery, and role-based access control across four permission tiers. Refresh token rotation with server-side session tracking.

👥

Guard Management

Full personnel lifecycle: self-registration with approval gate, attendance check-in/out, no-show detection, replacement requests, merit scoring, and punctuality records.

🔫

Firearms Inventory

End-to-end firearm tracking: inventory management, guard-to-firearm allocations, permit lifecycle with expiry alerts, and maintenance scheduling.

🚗

Armored Car Fleet

Fleet operations hub: vehicle inventory, driver assignments, predictive maintenance alerts, trip scheduling, and mission-to-car allocation.

📅

Scheduling & Calendar

Unified timeline view with shift creation, guard availability management, trip scheduling, mission assignment, and a full calendar dashboard.

🗺️

Real-Time Tracking

Live operational map powered by Leaflet + OpenStreetMap with WebSocket tracking stream, heartbeat ingestion, and a polling fallback for resilience.

🤖

AI Predictive Engine

Five built-in AI services: guard absence prediction, smart replacement suggestions, vehicle maintenance risk, incident severity classification, and incident summarization.

📊

Analytics Dashboard

KPI aggregation, utilization trends, performance reviews, merit score leaderboards, and predictive alert panels synthesizing multi-domain risk signals.

🛡️

Security & Audit

Audit trail for all write operations, presence tracking, rate limiting on auth endpoints, bcrypt password hashing, CSP headers, and CORS configuration.

Access Control

Role-Based Access Model

SENTINEL implements a four-tier RBAC model enforced at every API route via Axum middleware. Roles are normalized and validated case-insensitively.

SUPERADMIN

Superadmin

Full elevated system access. Operates the command center with unrestricted visibility across all modules and users.

  • Full user & role management
  • All analytics & audit logs
  • All firearms & fleet operations
  • AI predictions & alerts
  • System configuration
ADMIN

Administrator

Administrative operations including guard approvals, user management, scheduling, and asset management.

  • Approve guard registrations
  • Create & manage users
  • Manage schedules
  • Manage firearms & fleet
  • View analytics
SUPERVISOR

Supervisor

Operational supervision: monitors guard status, handles replacements, reviews attendance and incident reports.

  • Monitor guard attendance
  • Handle replacement requests
  • View & manage missions
  • View incidents & alerts
  • Access performance data
GUARD

Guard / Personnel

Daily operations workspace: schedule visibility, attendance check-in, firearm permit management, and support ticket submission.

  • View own schedule
  • Check-in / check-out
  • View firearm permit status
  • Submit support tickets
  • View own merit score

Authentication Flow

Register

Self-Registration (Guards)

Guards submit registration with Gmail address. Account created with pending status. Email verification code sent.

Verify

Email Verification

6-digit code emailed via Resend API. Guard submits code to activate account. Required before login is permitted.

Approval

Admin Approval Gate

Reviewer roles (superadmin / admin / supervisor) review pending registrations and approve or reject. Decision notification sent to guard.

Login

JWT Issuance

Login returns access JWT (7-day) + refresh token. Refresh tokens are persisted server-side and rotated on use. Claims include user ID and role.

Request

Authorized Requests

Every protected route validates the JWT, enforces RBAC permissions via authz.rs middleware, updates presence, and logs writes to the audit trail.

Predictive Intelligence

Built-In AI & Analytics Engine

Five purpose-built AI services running in the Rust backend provide real-time predictive insights across guard operations, fleet management, and incident response.

Guard Ops

Guard Absence Prediction

Analyzes historical attendance and punctuality patterns to predict which guards are likely to be absent on upcoming shifts — enabling proactive scheduling adjustments.

Guard Ops

Smart Replacement Suggestions

When a no-show is detected, the AI engine automatically scores and recommends the best available guard replacements based on availability, location, and merit score.

Fleet

Vehicle Maintenance Risk

Monitors mileage, last service dates, and trip frequency to predict when armored cars are approaching maintenance thresholds before breakdown occurs.

Incidents

Incident Severity Classification

Automatically classifies incoming incident reports by severity level (low / medium / high / critical) using pattern recognition to prioritize response queues.

Incidents

Incident Summarization

Generates concise natural-language summaries with extracted key phrases from verbose incident reports — reducing analyst reading time during high-volume alert periods.

Cross-Domain

Predictive Alerts Panel

Synthesizes all AI service outputs — permit expiry, maintenance overdue, no-show risk, and capacity issues — into a unified prioritized alert feed on the command center.

Developer Guide

Quick Start

Get SENTINEL running locally in minutes. The monorepo uses npm workspaces for frontend coordination and Cargo for the Rust backend.

Prerequisites

Node.js 18+
Rust (stable)
PostgreSQL 14+
Cargo

1 — Clone & Install

# Clone with submodules git clone --recurse-submodules https://github.com/dwaytu/Capstone-Main.git cd Capstone-Main # Install root and frontend dependencies npm install npm install --prefix DasiaAIO-Frontend

2 — Configure Environment

# DasiaAIO-Backend/.env DATABASE_URL=postgres://user:pass@localhost/sentinel JWT_SECRET=your-strong-secret-here ADMIN_CODE=your-admin-registration-code RESEND_API_KEY=re_xxxxxxxxxxxx # optional – email verification CORS_ORIGIN=http://localhost:5173 # DasiaAIO-Frontend/.env.local VITE_API_BASE_URL=http://localhost:3000

3 — Run Development Servers

# Terminal 1 – Backend (Rust) cd DasiaAIO-Backend cargo run # Terminal 2 – Frontend (Vite) npm run dev --prefix DasiaAIO-Frontend # → http://localhost:5173

4 — Build for Production

# From repository root: npm run build:web # Web distribution npm run build:desktop # Windows MSI + EXE (requires Tauri CLI) npm run build:android # Android APK (requires Android SDK) # Release all platforms with version tag: npm run release:all
