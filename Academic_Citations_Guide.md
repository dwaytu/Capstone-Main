## 1. Information Systems & Architecture

### Foundational Information Systems Theory

**Silver, M. S., Markus, M. L., & Beath, C. M. (1995).** "The Information Technology Interactive Model: A Foundation for the MBA Core Course." *MIS Quarterly*, 361-390.

- **Relevance**: Provides foundational framework for understanding information systems as integrated components of hardware, software, data, people, and procedures.

**Stair, R., & Reynolds, G. (2020).** *Principles of Information Systems* (14th ed.). Cengage. ISBN: 978-0-357-11252-6

- **Relevance**: Comprehensive coverage of integrated information system design, particularly the six-component framework: hardware, software, data, procedures, people, and networks.

**Jessup, L. M., & Valacich, J. S. (2008).** *Information Systems Today* (3rd ed.). Pearson Publishing. Glossary p. 416.

- **Relevance**: Defines information systems as formal, sociotechnical, organizational systems designed to collect, process, store, and distribute information.

### Systems Integration & Enterprise Applications

**Kroenke, D. M. (2008).** *Using MIS*. Prentice-Hall. ISBN: 978-0-13-713029-0

- **Relevance**: Addresses integration of multiple subsystems and enterprise-level information management practices applicable to security operations.

**O'Brien, J. A. (2003).** *Introduction to Information Systems: Essentials for the E-Business Enterprise*. McGraw-Hill.

- **Relevance**: Covers business processes, system integration, and organizational information technology alignment.

## 2. Web Services Architecture & REST APIs

### REST Architectural Style

**Fielding, R. T. (2000).** "Chapter 5: Representational State Transfer (REST)." In *Architectural Styles and the Design of Network-based Software Architectures* (Doctoral Dissertation). University of California, Irvine. Retrieved from http://www.ics.uci.edu/~fielding/pubs/dissertation/rest_arch_style.htm

- **Relevance**: Foundational work defining REST constraints, uniform interfaces, stateless operations, and hypermedia as the engine of application state (HATEOAS). Core reference for backend API design.

**Fielding, R. T. (2008).** "REST APIs must be hypertext driven." *roy.gbiv.com*. Retrieved from http://roy.gbiv.com/untangled/2008/rest-apis-must-be-hypertext-driven

- **Relevance**: Clarification of REST principles emphasizing proper API design standards.

**Richardson, L., & Ruby, S. (2007).** *RESTful Web Services*. O'Reilly Media. ISBN: 978-0-596-52926-0

- **Relevance**: Practical guide to REST API implementation, resource identification, and stateless operations.

**Erl, T., Carlyle, B., Pautasso, C., & Balasubramanian, R. (2012).** *SOA with REST: Principles, Patterns & Constraints for Building Enterprise Solutions with REST*. Prentice Hall. ISBN: 978-0-13-701251-0

- **Relevance**: Service-oriented architecture with REST constraints, scalability patterns, and enterprise integration practices.

## 3. Database Design & Management

### Database Design Methodology

**Teorey, T. J., Lightstone, S. S., & Nadeau, T. (2009).** *Database Design: Know It All* (1st ed.). Morgan Kaufmann Publishers.

- **Relevance**: Comprehensive coverage of database design phases including conceptual, logical, and physical modeling.

**Teorey, T., Lightstone, S., & Nadeau, T. (2005).** *Database Modeling & Design: Logical Design* (4th ed.). Morgan Kaufmann. ISBN: 0-12-685352-5

- **Relevance**: Detailed guidelines on entity-relationship modeling and logical schema development applicable to security data structures.

**Hernandez, M. (2013).** *Database Design for Mere Mortals: A Hands-On Guide to Relational Database Design* (3rd ed.). Addison-Wesley Professional. ISBN: 0-321-88449-3

- **Relevance**: Practical approach to normalization, data integrity, and designing maintainable database schemas.

### Data Integrity & Normalization

**Lightstone, S., Teorey, T., & Nadeau, T. (2007).** *Physical Database Design: The Database Professional's Guide to Exploiting Indexes, Views, Storage, and More*. Morgan Kaufmann. ISBN: 0-12-369389-6

- **Relevance**: Physical database implementation for performance optimization, indexing strategies, and data access patterns.

## 4. PostgreSQL & Relational Databases

### PostgreSQL Overview

**The PostgreSQL Global Development Group. (2025).** *PostgreSQL About*. Retrieved from https://www.postgresql.org/about/

- **Relevance**: Official documentation on PostgreSQL features, ACID compliance (since 2001), security mechanisms, extensibility, and proven reliability for enterprise systems.
- **Key Features Relevant to Project**:
  - Data integrity constraints (UNIQUE, NOT NULL, Primary Keys, Foreign Keys)
  - Transaction support with all SQL standard isolation levels
  - Multi-Version Concurrency Control (MVCC)
  - Robust access control system and role-based security
  - Write-ahead Logging (WAL) for reliability
  - Support for stored procedures and procedural languages

**The PostgreSQL Global Development Group. (2025).** *PostgreSQL Documentation*. Retrieved from https://www.postgresql.org/docs/

- **Relevance**: Authoritative technical reference for all PostgreSQL features and implementation details.

## Notes for Academic Use

1. **Verify Current Editions**: Check for the latest editions of textbooks when preparing final citations.

2. **Digital Object Identifiers (DOIs)**: Where available, include DOIs for improved citation precision.

3. **Access Information**: Document URLs and access dates for web-based sources.

4. **University Guidelines**: Confirm with your institution's citation style guide (APA, MLA, Chicago, etc.).

5. **Peer-Reviewed Sources**: Prioritize academic journals and conference proceedings over general websites.

6. **Professional Standards**: Reference IEEE and ACM publications for technical specificity.
